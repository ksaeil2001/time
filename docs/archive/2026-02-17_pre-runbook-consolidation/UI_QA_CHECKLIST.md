## 1) 공통 게이트
필수 통과 조건:
- [ ] 상태 배지/남은 시간/종료 시각을 3초 안에 읽을 수 있다.
- [ ] 사용자가 30초 안에 다음 행동(예약/취소/미루기)을 선택할 수 있다.
- [ ] Arm 확인 모달 없이 예약이 활성화되지 않는다.
- [ ] Final Grace에서 취소/미루기 액션이 유지된다.
- [ ] 단일 활성 예약 정책이 UI/트레이/이력에서 일관된다.

## 2) 화면별 3초 스캔 체크
### 대시보드
- [ ] `현재 상태` 카드에서 상태/남은 시간/종료 시각이 동시에 보인다.
- [ ] 활성 상태면 `지금 취소` 또는 `10분 미루기` 경로를 즉시 찾을 수 있다.
- [ ] 최근 이벤트가 중복 텍스트 없이 한 줄 요약 + 메타칩 구조로 보인다.

### 새 예약
- [ ] 모드(카운트다운/특정 시각/프로세스 감시)가 즉시 구분된다.
- [ ] 예상 종료가 절대시각+상대시간으로 보인다.
- [ ] 검증 실패 원인이 인라인으로 명확히 보인다.

### 활성 예약
- [ ] 취소/미루기 버튼이 스크롤 없이 보인다.
- [ ] 타임라인에서 현재 단계와 다음 단계를 3초 내 파악할 수 있다.
- [ ] 사용자 지정 미루기 입력 범위(1..1440)가 명확하다.

### 이력
- [ ] 필터/검색/정렬 상태를 즉시 파악할 수 있다.
- [ ] 행 하나에서 `무슨 일`, `언제`, `출처/모드`를 빠르게 읽을 수 있다.
- [ ] 상세 드로어 열기/닫기 시 포커스 복귀가 일관된다.

### 일반 설정
- [ ] `시뮬레이션 활성/실제 종료 활성` 상태가 3초 내 식별된다.
- [ ] 저장 정책(로컬 저장/보관 250)이 명확하다.
- [ ] 저장 성공/실패 피드백이 명확하다.

### 알림 설정
- [ ] 알림 차단 여부가 배너로 즉시 보인다.
- [ ] 사전 알림/최종 경고 값이 현재 값 기준으로 보인다.
- [ ] `알림 설정 열기` 복구 CTA가 동작한다.

### Google 연동
- [ ] 상태 배지(`연결됨/토큰 만료/오프라인/연결 안 됨`)가 즉시 보인다.
- [ ] 상태별 가능한 액션이 모호하지 않다.
- [ ] 에러 메시지와 재시도 경로가 있다.

### 도움말
- [ ] 취소/미루기 경로 안내가 첫 화면에서 보인다.
- [ ] FAQ가 한 줄 답변으로 스캔 가능하다.
- [ ] 오프라인 시 링크 비활성/경고가 명확하다.

## 3) 반응형 구간별 최소 보장
브레이크포인트:
- compact: 480 이하(토큰 기준 시뮬레이션)
- standard: 960 이상
- desktop: 1200 이상
- wide: 1440 이상

검증 항목:
- [ ] compact에서도 `Cancel/Snooze`가 스크롤 없이 보인다.
- [ ] standard에서 상태바 1행 스캔이 가능하다.
- [ ] desktop에서 본문 조작 우선순위가 유지된다.
- [ ] wide에서 우측 패널이 보조정보로만 동작하고 안전 액션을 가리지 않는다.

## 4) 접근성/상태 스타일 체크
키보드:
- [ ] `Ctrl/Cmd+N`으로 새 예약 이동
- [ ] `Esc`로 모달/상세 닫기
- [ ] History 테이블 행 Enter로 상세 열기

포커스:
- [ ] focus-visible 링이 모든 주요 인터랙션 요소에 보인다.
- [ ] Final Warning에서 포커스 트랩이 동작한다.
- [ ] Final Warning 종료 후 이전 포커스로 복귀한다.

대비/비활성:
- [ ] disabled 버튼이 클릭 불가 상태를 시각적으로 명확히 보여준다.
- [ ] 색상 외 텍스트/아이콘으로 상태를 구분한다.
- [ ] 시간/숫자 표시는 tabular로 정렬된다.
